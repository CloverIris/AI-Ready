# AI-Ready UX/UI 设计指南

## 1. 设计理念

### 1.1 核心原则

1. **像安装 Windows 一样简单**：OOBE 引导体验，分步骤进行
2. **渐进式披露**：基础用户看简化界面，高级用户可展开详细选项
3. **即时反馈**：所有操作有进度指示，长时间操作可取消
4. **教育融入**：在合适时机解释技术概念（PATH、CUDA、VENV）

### 1.2 设计语言

- **遵循 Fluent Design 2**：Mica/Acrylic 材质，圆角，流畅动画
- **色彩系统**：
  - 主色：Windows 蓝 `#0078D4`
  - 成功：绿色 `#107C10`
  - 警告：橙色 `#FFB900`
  - 错误：红色 `#D13438`
- **字体**：Segoe UI Variable

## 4. 交互细节

### 4.1 通知与提示

| 场景 | 类型 | 内容 |
|------|------|------|
| 工作流安装完成 | 成功 Toast | "Ollama 安装完成！点击启动" |
| 服务器断开 | 警告 Banner | "与 AutoDL-南京 的连接已断开，正在重试..." |
| 磁盘空间不足 | 错误 Dialog | "磁盘空间不足，需要至少 20GB 可用空间" |
| 首次看到 PATH 设置 | 教育 Tip | "💡 PATH 是系统查找可执行文件的路径列表" |

## 5. 响应式考虑

由于 WinUI 3 应用，主要考虑窗口大小适配：

| 宽度 | 布局 |
|------|------|
| > 1200px | 三栏布局：导航 + 主内容 + 详情 |
| 800-1200px | 两栏布局：导航 + 主内容 |
| < 800px | 单栏：导航折叠为汉堡菜单 |
